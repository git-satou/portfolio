- /etc/nginx/conf.d/vhost.conf 

```
server {
  listen 80;
  server_name example.com www.example.com;
  access_log /var/log/nginx/example.com-access.log combined;
  error_log /var/log/nginx/example.com-error.log warn;

  rewrite ^(.*)$ https://$host$request_uri redirect;
}

server {
  listen 443 ssl;
  server_name example.com www.example.com;
  access_log /var/log/nginx/example.com-ssl-access.log combined;
  error_log /var/log/nginx/example.com-ssl-error.log warn;

  ssl_session_timeout 1d;
  ssl_session_cache shared:SSL:50m;
  ssl_session_tickets on;
  ssl_protocols TLSv1 TLSv1.1 TLSv1.2;
  ssl_ciphers HIGH:!aNULL:!MD5;
  ssl_prefer_server_ciphers on;
  ssl_certificate ssl/server.crt;
  ssl_certificate_key ssl/server.key;

  root /var/www/html;

  location / {
    index index.php index.html index.htm;
  }

  location ~ \.php$ {
    fastcgi_split_path_info ^(.+\.php)(/.+)$;
    fastcgi_pass   unix:/run/php-fpm/www.sock;
    fastcgi_index  index.php;
    include        fastcgi_params;
    fastcgi_param  SCRIPT_FILENAME $document_root$fastcgi_script_name;
    fastcgi_param  PATH_INFO $fastcgi_path_info;
    fastcgi_param  PATH_TRANSLATED $document_root$fastcgi_path_info;
  }

  location ~ /\.ht {
    deny  all;
  }

  location ~ /\.user\.ini {
    deny  all;
  }
}
```

## 上位にロードバランサーがある場合はhttpセクションに下記を追記
※set_real_ip_fromはバランサーのIP

```
    set_real_ip_from {IP/CIDR};
    real_ip_header     X-Forwarded-For;
```

## 上位にロードバランサーがあり、さらに上位にcloudfrontなどがある場合はhttpセクションに下記を追記
※cloudfrontの場合はIPが可変なので、0.0.0.0/0;にするしかない

```
    set_real_ip_from 0.0.0.0/0;
    real_ip_header     X-Forwarded-For;
    real_ip_recursive on;
```


## その他
この時にリロード処理が下記のような場合、/var/log/nginxの権限を755にしないと正常にログがローテートされない。
最近のレポジトリだとおそらく権限は755になっているはずだが、ちょっと古いものは注意


```
    postrotate
        /bin/kill -USR1 `cat /run/nginx.pid 2>/dev/null` 2>/dev/null || true
```
